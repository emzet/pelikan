1. Algoritmizácia:

"Bonusové body" získajú riešenia ktoré:

  • (!) sú riešené funkcionálnym stýlom programovania (pure funkcie, immutabilné hodnoty, rekurzia atď)
  • budú napísané v TypeScripte
  • budú sprevádzané jednoduchými unit testami (voľný výber testovacieho frameworku)

      • 1.1 Simple arrays

      Vytvorte funkciu vyhovujúcu nasledujúcemu zápisu:

                      getLengthOfMissingArray(arrayOfArrays: any[]): number

      vstupom bude pole polí, napr.

                      [
                  [4, 6, 8, 1],
                  [2, 1],
                  [899, null, 5, 0],
                  [17]
                ]
      • Pri zoradeni tohto poľa je zrejmé, že dĺžky polí v ňom majú postupnosť, avšak jedno pole chýba.
      • Funkcia má vrátiť dĺžku tohto poľa (v tomto prípade 3).
      • Pokiaľ je "hlavné" pole prázne alebo null, funkcia vráti 0.
      • Počítajte s tým, že vždy bude chýbať práve jedno pole ktorého dĺžka bude medzi dĺžkami polí v poli

      Príklady

                      getLengthOfMissingArray([
                  [ 1, 2 ],
                  [ 4, 5, 1, 1 ],
                  [ 1 ],
                  [ 5, 6, 7, 8, 9 ]
                ]); // vráti 3
              
                      getLengthOfMissingArray([
                  [ 5, 2, 9 ],
                  [ 4, 5, 1, 1 ],
                  [ 1 ],
                  [ 5, 6, 7, 8, 9 ]
                ]); // vráti 2
              
                      getLengthOfMissingArray([ [ ] ]); // vráti 0
              
                      getLengthOfMissingArray([ [ null ], [ null, null, null ] ]); // vráti 2
              
                      getLengthOfMissingArray([
                  [ 'a', 'a', 'a' ],
                  [ 'a', 'a' ],
                  [ 'a', 'a', 'a', 'a' ],
                  [ 'a' ],
                  [ 'a', 'a', 'a', 'a', 'a', 'a']
                ]); // vráti 5

      1.2. Simple strings

                      decipherThis(str: string): string

      Funkcia odšifruje každé slovo zvlášť v zašifrovej správe. Podmienky:

      • Prvé písmeno korešponduje kú kódu ASCII znaku (case sensitive)
      • Druhé písmeno treba zameniť s posledným písmenom
      • Posledné písmeno treba vymeniť s druhým písmenom
      • Pokiaľ má správa iba jedno písmeno, zostane nezmenená
      • Pokiaľ má správa iba dve písmená, je iba potrebné prekonvertovat kód ASCII znaku na písmeno
      • Žiadne prísne edge cases - správa je bez špeciálnych znakov atď.

      Príklady

                      decipherThis('72olle 103doo 100ya'); // vráti 'Hello good day'
              
                      decipherThis('82yade 115te 103o'); // vráti 'Ready set go'

      2. Funkcionálne programovanie:

      2.1 ifs are so 2005

                      function getGenderByOrderType(order: any): string {
                  var gender: string;

                  if (order.type == 'flight') {
                    if (order.gender == 'F') {
                      gender = 'FEMALE';
                    }
                    else if (order.gender == 'M') {
                      gender = 'MALE';
                    }
                    else {
                      gender = '';
                    }
                  }
                  else if (order.type == 'vacation') {
                    if (order.gender == 'F') {
                      gender = 'WOMAN';
                    }
                    else if (order.gender == 'M') {
                      gender = 'MAN';
                    }
                    else {
                      gender = '';
                    }
                  }
                  else {
                    gender = '';
                  }

                  return gender;
                }

      Uvedenú funkciu naštudujte a analyzujte čo robí.
      Cieľom je zjednodušiť ju, aby bola čitateľnejšia, deskriptívnejšia, lahšie rozšíriteľnejšia, teda dosiahnuť zápis ktorý:

      • nevyužíva if, else, else if, ani switch.
      • nevyužíva premennú var, ale iba const (jedna by mala stačiť)
      • popíšte, aký má tento zápis výhody a nevýhody, kedy je vhodný a kedy vyslovene nevhodný
      • predpokladajme, že argument order nebude null ani undefined.

      2.2 simple FP

      Analyzujte a upravte nasledujúcu funkciu:

                      function divideSmallestNumber(numbers: number[]): string {
                  var evenNumber;
                  var dividedNumber;

                  numbers = numbers.sort(function(a, b) { return a - b; })

                  for (var i = 0; i < numbers.length; i++) {
                    if (numbers[i] % 2 === 0) {
                      evenNumber = numbers[i];
                    }
                  }

                  dividedNumber = evenNumber / 2;

                  if (dividedNumber < 10) {
                    dividedNumber = '0' + dividedNumber;
                  }

                  return '' + dividedNumber;
                }
      • na fukncionálny zápis, pomocou funkcií map, reduce atď.
      • premenné by nemali byť potrebné (jeden const by mal stačiť), pracujte so vstupným poľom ako by bolo immutabilné
      • uprednostnite ternárny operátor pred if / else blokmi
      • možete počítať s tým, že v poli nebude nula ani záporné čísla a bude mať aspoň jedno párne číslo

      2.3 object map

      Napíšte funkciu ktorá premapuje hodnoty v objekte na uppercase, pokiaľ nezačínajú číslom

                      const person = {
                  'firstName': 'John',
                  'lastName': 'Doe',
                  'height': '185 cm',
                  'kids': '2 sons',
                  'from': 'Philadelphia'
                };

                const upperCaseIfNotStartsWithNumber = (object: any): any => {
                  ...
                };

                upperCaseIfNotStartsWithNumber(person);
                /*
                vráti:
                {
                  'firstName': 'JOHN',
                  'lastName': 'DOE',
                  'height': '185 cm',
                  'kids': '2 sons',
                  'from': 'PHILADELPHIA'
                }
                */

Vypracované riešenia dodajte formou ktorá vám najviac vyhovuje a podľa toho či ste sa rozhodli ich vypracovať v TS a teda s task runnermi na kompiláciu TS, poprípade testami, alebo v jednoduchšej verzii na stránky typu jsfiddle.
